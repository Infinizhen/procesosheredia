# Security Policy

This is a **static website with no backend** — no server, no database, no user
accounts, and no personal data collected from visitors. The attack surface is
small by design. Still, if you find something, please report it responsibly.

## Reporting a vulnerability

**Please do not open a public issue for security problems.**

Use **GitHub's private vulnerability reporting** instead:

> Go to the **[Security](https://github.com/Infinizhen/procesosheredia/security)**
> tab of this repository → **Report a vulnerability**.

That keeps the report private until it's resolved. If that channel isn't available
to you, you can reach out through the contact link on
[procesosheredia.com](https://procesosheredia.com).

Please include:

- what the issue is and where (file, URL, or request),
- how to reproduce it, and
- the impact, as you see it.

There's no bug-bounty program — just genuine thanks for reporting responsibly.

## Scope

In scope: this repository's source and the deployed site at
`procesosheredia.com`.

Out of scope: third-party platforms the site merely links to or embeds (Spotify,
Instagram, TikTok, the hosting/CDN provider). Report those to the respective
vendor.

## Supported versions

Only the **currently deployed** version (the live site) is maintained. There are no
released versions or branches to back-port fixes to.
